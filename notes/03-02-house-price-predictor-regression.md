# 03.02 — Use Case: House Price Predictor — Regression

## Цель урока

Сформулировать прогнозирование стоимости жилья как задачу машинного обучения и понять основные этапы работы модели.

## Постановка задачи

Модель получает характеристики объекта недвижимости и прогнозирует его стоимость.

Примеры входных признаков (`features`):

- площадь;
- количество комнат и ванных комнат;
- местоположение;
- год постройки или возраст здания;
- состояние объекта.

Целевая переменная (`target`) — цена жилья. Это задача `regression`, потому что результатом является числовое значение, а не заранее определённый класс.

## Основные термины

- `Feature` — характеристика объекта, подаваемая модели на вход.
- `Target variable` — значение, которое модель должна предсказать.
- `Feature engineering` — получение новых признаков из исходных данных, например возраста здания из года постройки.
- `Training data` — исторические примеры с известными признаками и ценами.
- `Algorithm` — метод поиска закономерностей в данных.
- `Model` — результат обучения алгоритма на конкретных данных.
- `Training` — процесс обучения модели на исторических данных.
- `Inference` — получение предсказания от уже обученной модели на новых данных.

## Общий процесс

```text
Historical data + algorithm
              ↓
           training
              ↓
        trained model
              ↓
New house features → inference → predicted price
```

Модель изучает связи между ценой и характеристиками проданных объектов, а затем применяет найденные закономерности к новому объекту.

## Связь с MLOps и DevOps

Поведение ML-системы зависит не только от кода и конфигурации, но также от обучающих данных, признаков, алгоритма и версии модели. MLOps должен сделать обучение, проверку и использование модели воспроизводимыми и управляемыми.

Понимание разницы между `training` и `inference` помогает определить разные требования этих этапов к ресурсам, данным и эксплуатации.

## Важные замечания

- Упомянутая в уроке `logistic regression` предназначена для classification и не подходит для непосредственного прогноза непрерывной цены.
- Стоимость квадратного фута, рассчитанная из известной цены, нельзя использовать для предсказания этой же цены: это `target leakage`.
- Возраст здания, рассчитанный из года постройки, является корректным примером feature engineering.

## Проверка понимания

- Прогноз цены относится к regression, потому что target — числовое значение, а не класс жилья.
- Training обучает модель на исторических данных; inference использует обученную модель для получения нового предсказания.

## Результат урока

Сформулирован ML use case: обучить regression model на исторических данных о недвижимости и предсказывать стоимость нового объекта по его характеристикам. Практического кода в уроке нет.
